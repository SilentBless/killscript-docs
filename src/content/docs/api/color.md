`Color` хранит красный, зелёный, синий и альфа-компонент цвета. Он используется в UI, `ImGui`, визуальных эффектах и других API, принимающих цвет.

## Быстрый пример

```lua
local cyan = Color.new(0.2, 0.8, 1)
local translucentRed = Color.new(1, 0, 0, 0.5)

print(cyan.a) -- 1
```

Если передать только RGB, конструктор автоматически задаёт `a = 1`.

## Создание цвета

```lua
Color.new(r: number, g: number, b: number): Color
Color.new(r: number, g: number, b: number, a: number): Color
```

| Аргумент | Тип | Описание |
|---|---|---|
| `r` | `number` | Красный компонент. |
| `g` | `number` | Зелёный компонент. |
| `b` | `number` | Синий компонент. |
| `a` | `number` | Альфа-компонент. Если не указан, равен `1`. |

## Компоненты

| Свойство | Тип | Доступ | Описание |
|---|---|---|---|
| `r` | `number` | `get/set` | Красный компонент. |
| `g` | `number` | `get/set` | Зелёный компонент. |
| `b` | `number` | `get/set` | Синий компонент. |
| `a` | `number` | `get/set` | Альфа-компонент. |

Компоненты можно менять после создания:

```lua
local color = Color.new(1, 1, 1)
color.g = 0.5
color.a = 0.75
```

:::caution[Значения не ограничиваются автоматически]
`Color` сохраняет числа за пределами `0..1`. Например, `Color.new(-0.5, 1.5, 2, 3)` создаётся без ошибки и без приведения компонентов к допустимому диапазону. Если принимающий API ожидает обычный цвет, ограничивайте значения самостоятельно.
:::

## Частые ошибки

### Цвет задан числами от 0 до 255

Обычная запись использует значения от `0` до `1`. Для цвета RGB `128, 200, 255` разделите компоненты на `255`:

```lua
local color = Color.new(128 / 255, 200 / 255, 1)
```

### Неожиданная непрозрачность

Трёхкомпонентный конструктор задаёт `a = 1`. Передайте четвёртый аргумент, если нужен другой альфа-компонент.
