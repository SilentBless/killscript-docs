## Справочник API

На страницах указана не только сигнатура, но и путь обработки. Различайте:

- **значение** (`Vector3`, `Color`) — только хранит данные, пока его не передали другому API;
- **query/result** (`Raycast`, `GetAll`, структуры событий) — наблюдает или описывает уже рассчитанный результат;
- **live object** (`Agent`, `Item`, `Entity`) — ссылается на существующий игровой объект и может стать недоступным;
- **config object** (`ConfigItem*`) — описывает правила типа объекта, но не является живым предметом;
- **command** — передаёт изменение конкретной подсистеме; возвращаемое `nil` не всегда означает, что сервер принял результат.

В начале каждой страницы ищите, какая система владеет данными, где выполняется вызов и кто показывает результат.
