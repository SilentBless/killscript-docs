---
title: Команды сервера
description: Актуальные консольные команды пользовательского сервера KILLSCRIPT и их аргументы.
---

Откройте консоль клавишей `~` и введите команду без дополнительного префикса. Консоль показывает только разрешённые текущему игроку команды.

## Управление матчем — Host

| Команда | Назначение |
|---|---|
| `start` | Начать пользовательский матч из разминки. |
| `restart` | Перезагрузить текущую карту. |
| `reload` | Перезагрузить игровую базу/подключённую конфигурацию. |
| `map-list` | Показать технические имена доступных карт. |
| `load-map <scene_name>` | Переключиться на карту по имени из `map-list`. |

Пример:

```text
map-list
load-map Castle
```

Используйте точное имя, которое вернул `map-list`.

## Агенты и предметы — Host

| Команда | Назначение |
|---|---|
| `list-agents` | Показать ID всех агентов матча. |
| `heal [agent_id]` | Вылечить указанного агента или своего без аргумента. |
| `kill [agent_id]` | Убить указанного агента или своего без аргумента. |
| `kill-team <team_id>` | Убить всех агентов команды. |
| `give-item <item_name>` | Выдать предмет по его техническому имени. |

Примеры:

```text
list-agents
heal 3
kill-team 2
give-item LightPistol
```

Имена предметов можно уточнить в [GameConfig](../../api/game-config/) или по текущему игровому содержимому.

## Игроки — Host

| Команда | Назначение |
|---|---|
| `list-players` | Показать ID подключённых игроков. |
| `kick <player_id>` | Отключить игрока от сервера. |

ID игрока и ID агента — разные значения. Для `kick` используйте результат `list-players`, а для `heal` и `kill` — `list-agents`.

## Личные и командные команды

| Команда | Назначение |
|---|---|
| `suicide` | Убить собственного агента. |
| `spectate` | Переключить режим наблюдателя. |
| `change-team` | Перейти в противоположную команду. |
| `change-agent <id>` | Пересесть в доступного агента по ID. |
| `version` | Показать версию серверного приложения. |

Доступ к `spectate`, `change-team` и `change-agent` зависит от режима матча. На пользовательском сервере консоль сама показывает фактически разрешённые команды.

## Если команда не сработала

- проверьте написание и количество аргументов;
- получите свежий ID через `list-agents` или `list-players`;
- убедитесь, что вы являетесь host;
- для карты используйте результат `map-list`;
- посмотрите ответ команды и фильтр ошибок в консоли.

:::note
Диагностические admin-команды намеренно не включены в пользовательское руководство: они недоступны обычному владельцу сервера или не работают в release-сборке.
:::

Создание сессии, права и подключение конфигурации описаны на странице [пользовательского сервера](../custom-server/).
