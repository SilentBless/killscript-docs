# Проверка API KILLSCRIPT

Эта директория содержит внутренний план аудита и исходники диагностических модулей. Файлы из `validation/` не публикуются на сайте документации.

## Принцип проверки

Старая русская документация используется только как перечень кандидатов. Для каждого элемента API отдельно проверяются:

1. наличие в заявленном контексте;
2. фактический тип;
3. чтение свойства;
4. запись свойства с последующим чтением и восстановлением значения;
5. успешный сценарий метода;
6. граничные значения и ошибки, если их можно проверить безопасно;
7. наблюдаемое поведение в игре;
8. минимальный пример, запускаемый отдельно от диагностического стенда.

Проверка доступа и проверка поведения — разные этапы. Успешная запись того же значения подтверждает наличие setter, но не доказывает, что изменение применяется. Поэтому изменяемые свойства дополнительно тестируются контролируемым значением с восстановлением исходного состояния.

## Контексты

- `ApiValidationClient` запускает проверки обычного клиентского модуля.
- `ApiValidationReflex` проверяет клиентскую и серверную части Reflex-модуля.

Каждый батч должен быть небольшим. Ошибка Lua bridge в одном API не должна мешать проверке остальных разделов.

## Публикация

Раздел переносится в `src/content/docs/` только после завершения всего логического батча. Сначала создаётся русская страница, затем английская страница с тем же относительным путём.
